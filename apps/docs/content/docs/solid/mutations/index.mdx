---
title: Mutations
description: Solid mutation factories for Better Auth write endpoints.
---

Solid mutations preserve shared mutation keys and pass `throw: true` so failures surface through standard Solid Query error handling.

Named Solid mutations expose options factories for `createMutation`. Many named mutations also expose `use*` hooks for component code. Use `useAuthMutation` as the generic escape hatch when an endpoint does not have a named helper.

```tsx
import { signInEmailOptions } from "@better-auth-ui/solid"
import { createMutation } from "@tanstack/solid-query"

const signIn = createMutation(() => signInEmailOptions(authClient))
```

## Error handling

Every named factory wires `throw: true` into `fetchOptions`, so the mutation rejects with a `BetterFetchError` on failure instead of resolving with `{ error }`.

## Cache side effects

Core auth factories stay explicit: invalidate or refetch with shared keys after a successful mutation. Organization hook mutations additionally attach `meta.invalidates` / `meta.awaits`, and `AuthProvider` installs the Solid mutation invalidator that consumes that metadata.

```ts
import { authQueryKeys } from "@better-auth-ui/core"

await queryClient.invalidateQueries({ queryKey: authQueryKeys.session })
```

## Escape hatch

For write-style endpoints without a purpose-built factory or hook, use `useAuthMutation` in components or `authMutationOptions` where Solid Query takes mutation options.

## UI boundary

The Solid package does not render toast UI. Attach success/error handling in your app, or use installed Zaidan components that already integrate with the app's toast surface.
