```tsx title="routes/__root.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/__root.tsx
```

</Step>
<Step>
### Create the Auth Page

Create a dynamic auth page that renders the appropriate authentication view based on the path. Validate the segment against `resolveViewPaths(authPlugins).auth` so unknown paths return a 404.

```tsx title="routes/auth/$path.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/auth/$path.tsx
```

The resolved `auth` object contains every built-in path (`sign-in`, `sign-up`, `sign-out`, `forgot-password`, `reset-password`) plus one entry for each plugin in `authPlugins` that contributes an auth view. If you haven't registered any plugins, call `resolveViewPaths()` with no arguments to fall back to the built-in defaults.

</Step>
<Step>
### Create the Settings page

Create a dynamic settings route that renders the `Settings` component for the URL segment. Validate the segment against `viewPaths.settings` so unknown paths return a 404.

```tsx title="routes/settings/$path.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/settings/$path.tsx
```

The built-in `viewPaths.settings` object contains `account` and `security`. When plugins contribute their own settings views (e.g. `organizationPlugin()` adding `organizations`), swap the import for `resolveViewPaths(authPlugins).settings` just like the auth route above.

</Step>
</Steps>

## Protecting Routes

Use the `useAuthenticate` hook to protect routes and access session data.

```tsx title="routes/dashboard.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/dashboard.tsx
```

The `useAuthenticate` hook will automatically redirect unauthenticated users to the sign-in page.

## Registering Plugins

Plugins extend the UI with additional auth flows, views, and slot components. The recommended pattern is a single `lib/auth-plugins.ts` module that defines every plugin you want enabled:

```tsx title="lib/auth-plugins.ts" file=<rootDir>/../../examples/start-heroui-example/src/lib/auth-plugins.ts
```

Pass the array into `AuthProvider` via the `plugins` prop, and into `resolveViewPaths()` wherever you need to validate or look up plugin-contributed paths outside of React (route loaders, server handlers, middleware).

To add another plugin, import its factory and append it to the array:

```tsx title="lib/auth-plugins.ts"
import {
  magicLinkPlugin,
  multiSessionPlugin,
  passkeyPlugin
} from "@better-auth-ui/heroui/plugins"

export const authPlugins = [
  magicLinkPlugin(),
  multiSessionPlugin(),
  passkeyPlugin()
]
```

The plugin's localization, view paths, views, and slot components (auth buttons, security cards, account cards, user menu items) are all wired up automatically â€” no other file needs to change. Install each plugin's matching client on your `authClient` too (e.g. `magicLinkClient()` on `createAuthClient` alongside `magicLink()` on the Better Auth server).
