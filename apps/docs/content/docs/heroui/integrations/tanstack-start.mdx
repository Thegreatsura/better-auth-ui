## Integration

<Steps>
<Step>
### Configure AuthProvider

Configure `AuthProvider` with TanStack Router's navigation. Register any plugins you want enabled by passing their factories inline to the `plugins` prop.

```tsx title="components/providers.tsx" file=<rootDir>/../../examples/start-heroui-example/src/components/providers.tsx
```

The `navigate` prop integrates with TanStack Router's navigation system. The `navigate` prop accepts `{ to, replace }` options to handle both push and replace navigation.

</Step>
<Step>
### Update the Root Route

Wrap your application with the `Providers` component in your root route.

```tsx title="routes/__root.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/__root.tsx
```

</Step>
<Step>
### Create the Auth Page

Create a dynamic auth page that renders the appropriate authentication view based on the path. For plugin-contributed paths (e.g. `/auth/magic-link` from `magicLinkPlugin()`), spread the plugin's `viewPaths.auth` into the validation set:

```tsx title="routes/auth/$path.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/auth/$path.tsx
```

The built-in `viewPaths.auth` object contains `sign-in`, `sign-up`, `sign-out`, `forgot-password`, and `reset-password`. Spread each plugin's `viewPaths.auth` to add its segments. If you aren't using any plugins, you can drop the spread and use `viewPaths.auth` directly.

</Step>
<Step>
### Create the Settings page

Create a dynamic settings route that renders the `Settings` component for the URL segment. Validate the segment against `viewPaths.settings` so unknown paths return a 404.

```tsx title="routes/settings/$path.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/settings/$path.tsx
```

The built-in `viewPaths.settings` object contains `account` and `security`. When a plugin contributes its own settings views (e.g. `organizationPlugin()` adding `organizations`), spread its `viewPaths.settings` the same way as the auth route above.

</Step>
</Steps>

## Protecting Routes

Use the `useAuthenticate` hook to protect routes and access session data.

```tsx title="routes/dashboard.tsx" file=<rootDir>/../../examples/start-heroui-example/src/routes/dashboard.tsx
```

The `useAuthenticate` hook will automatically redirect unauthenticated users to the sign-in page.

## Adding More Plugins

To add another plugin, import its factory and append it to the `plugins` array on `AuthProvider`:

```tsx title="components/providers.tsx"
import {
  magicLinkPlugin,
  multiSessionPlugin,
  passkeyPlugin
} from "@better-auth-ui/heroui/plugins"

<AuthProvider
  plugins={[magicLinkPlugin(), multiSessionPlugin(), passkeyPlugin()]}
  /* ... */
/>
```

Then spread each plugin's view paths in the routes that need to validate them:

```tsx title="routes/auth/$path.tsx"
const authViewPaths = {
  ...viewPaths.auth,
  ...magicLinkPlugin().viewPaths?.auth,
  ...passkeyPlugin().viewPaths?.auth
  // ...etc
}
```

Install each plugin's matching client on your `authClient` too (e.g. `magicLinkClient()` on `createAuthClient` alongside `magicLink()` on the Better Auth server).
