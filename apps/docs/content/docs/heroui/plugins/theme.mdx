The theme plugin adds theme selection to your authentication UI. Users can switch between system, light, and dark themes from the user button dropdown and account settings.

## Setup

<Steps>
<Step>
### Register the UI plugin

Pass `themePlugin()` to `<AuthProvider>` with your theme management functions:

```tsx title="components/providers.tsx"
import { AuthProvider } from "@better-auth-ui/heroui"
import { themePlugin } from "@better-auth-ui/heroui/plugins" // [!code highlight]

<AuthProvider
  authClient={authClient}
  plugins={[
    themePlugin({ // [!code highlight]
      theme: currentTheme, // [!code highlight]
      themes: ["system", "light", "dark"], // [!code highlight]
      setTheme: (theme) => setTheme(theme) // [!code highlight]
    }) // [!code highlight]
  ]}
>
  {children}
</AuthProvider>
```

The plugin reads the theme config passed to `themePlugin()` to get the current `theme`, `setTheme` function, and available `themes` list.

</Step>
<Step>
### Wire up your theme library

The plugin is theme-library agnostic. Connect it to your preferred theme management solution (e.g., [next-themes](https://github.com/pacocoursey/next-themes)):

```tsx title="app/providers.tsx"
import { AuthProvider } from "@better-auth-ui/heroui"
import { themePlugin } from "@better-auth-ui/heroui/plugins" // [!code highlight]
import { ThemeProvider, useTheme } from "next-themes"
import { ReactNode } from "react"

import { authClient } from "@/lib/auth-client"

function Providers({ children }: { children: ReactNode }) {
  const router = useRouter()

  return (
    <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
      <AuthProvider
        authClient={authClient}
        plugins={[
          themePlugin(useTheme()))  // [!code highlight]
        ]}
      >
        {children}
      </AuthProvider>
    </ThemeProvider>
  )
}
```

</Step>
</Steps>
