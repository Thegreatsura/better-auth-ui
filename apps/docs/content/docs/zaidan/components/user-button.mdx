## Usage

Install `user-button` when you need the signed-in account menu in a Solid route or header. It reads the active session from `AuthProvider`, shows sign-in/sign-up actions when unauthenticated, and renders settings, plugin-contributed user menu items, theme, and sign-out entries when a user is signed in.

```tsx
import { UserButton } from "@/components/auth/user/user-button"

export function HeaderActions() {
  return <UserButton size="icon" align="end" />
}
```

Use the default size when the trigger should show the avatar, user label, and secondary label together.

```tsx
import { UserButton } from "@/components/auth/user/user-button"

export function AccountMenu() {
  return <UserButton />
}
```

The menu uses the copied theme item from the Zaidan registry entry and the route paths from your `AuthProvider` config. When a plugin contributes `userMenuItems`, `UserButton` renders them in the authenticated dropdown. With Multi Session enabled, that means the user menu can inject the multi-session submenu without replacing the built-in settings, theme, or sign-out actions.

## Installation

```npm
npx zaidan add https://better-auth-ui.com/r/solid/user-button.json
```

### Parity classification

`links` and `hideSettings` are Solid-adapted matches for the shadcn user menu contract. Zaidan exposes the styling hook as Solid `class` to follow local Solid conventions.

## Installed files

- `src/components/auth/user-button.tsx`
- `src/components/auth/user/user-button.tsx`
- `src/components/auth/user/user-avatar.tsx`
- `src/components/auth/user/user-view.tsx`
- `src/components/auth/theme/theme-toggle-item.tsx`

After install, point the auth and settings links at your TanStack Router paths, then decide whether to keep the icon trigger or the full account row and which plugins should contribute extra authenticated user menu items.
