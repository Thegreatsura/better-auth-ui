`OrganizationSwitcher` reads the current user's organizations, shows the active organization or personal account, and navigates with the slug-aware Organization runtime.

## Usage

```tsx
import { OrganizationSwitcher } from "@/components/auth/organization/organization-switcher"

<header>
  <OrganizationSwitcher />
</header>
```

Use `setActive` to override the default slug navigation or session mutation behavior:

```tsx
<OrganizationSwitcher
  setActive={(organization) => {
    if (organization) {
      console.log("Switch to", organization.slug)
      return
    }

    console.log("Switch to personal account")
  }}
/>
```

You can also hide personal-account or settings entries and provide a custom trigger:

```tsx
<OrganizationSwitcher
  hidePersonal
  hideSettings
  hideSlug={false}
  trigger={<button type="button">Organizations</button>}
/>
```

## Installation

## Notes

The switcher uses `useListOrganizations`, `useActiveOrganization`, and `useSetActiveOrganization` from `@better-auth-ui/solid`. In slug-aware routes it navigates to `/organization/$slug/settings`; outside slug mode it preserves the legacy set-active mutation flow.

Focused switcher parity includes `setActive`, `hidePersonal`, `hideSettings`, `hideSlug`, and custom trigger support. create-dialog parity is deferred; users can create organizations from `/settings/organizations` in this slice.
