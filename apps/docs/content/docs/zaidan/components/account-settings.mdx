## Usage

Use `AccountSettings` inside your settings shell when the active tab is account.

```tsx
import { AccountSettings } from "@/components/auth/settings/account/account-settings"

export function AccountTab() {
  return <AccountSettings />
}
```

The component composes the base account cards shipped by the Solid registry entries—profile, email, and theme appearance—and then renders plugin-contributed `accountCards`. With Multi Session enabled, that means `<ManageAccounts />` is injected by the registered plugin instead of being hardcoded into the settings stack.

## Installation

```npm
npx zaidan add https://better-auth-ui.com/r/solid/account-settings.json
```

### Parity classification

Zaidan exposes the wrapper styling hook as Solid `class`, adapting the shadcn `className` prop to the local Solid convention. Behavior remains configured through `AuthProvider`, Better Auth plugins, and installed child files.

## Plugin gates

- `theme` powers the appearance card.
- `multi-session` can inject account switching through plugin-contributed `accountCards`.
- `delete-user` is a related account/security registry entry; expose the danger-zone UI only when the Better Auth delete-user plugin is configured.

## Installed files

- `src/components/auth/settings/account/account-settings.tsx`
- `src/components/auth/settings/account/appearance-settings.tsx`

After install, pair it with `user-profile`, `change-email`, `theme`, and whichever plugins contribute extra account cards in your app.
