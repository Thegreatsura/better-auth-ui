`Organization` is the minimal route shell for `/organization/$slug/$path`. It resolves the active organization for the slug and renders settings or people panels that point to the Solid organization APIs.

## Usage

```tsx
import { Organization } from "@/components/auth/organization/organization"

<Organization path="settings" slug="acme" />
```

## Installation

```bash
zaidan add https://better-auth-ui.com/r/solid/organization.json
```

## Deferred UI

After install, wire `organization()` on the server, `organizationClient()` on the client, and the local `organizationPlugin()` in your provider before exposing the route.

The route shell is intentionally small. Full profile forms, member tables, invitation tables, logo upload, and destructive dialogs are deferred in this PR slice and should be built from the Solid organization query/mutation APIs.
