`Organization` is the slug-aware route shell for `/organization/$slug/$path`. It resolves the active organization for the slug, renders profile/settings UI for the `settings` route, preserves plugin-contributed organization cards, and renders the People shell for the `people` route.

## Usage

```tsx
import { Organization } from "@/components/auth/organization/organization"

<Organization path="settings" slug="acme" />
```

## Installation

```npm
npx zaidan add https://better-auth-ui.com/r/solid/organization.json
```

## Notes

After install, wire `organization()` on the server, `organizationClient()` on the client, and the local `organizationPlugin()` in your provider before exposing the route.

The settings route includes organization profile editing, slug editing, logo upload/delete, and plugin-contributed `organizationCards`. The people route renders `OrganizationPeople` with a basic members list and an invitations shell section. Advanced members tables, invitations tables, roles, and destructive dialogs remain follow-up slices.
