`OrganizationPeople` is the shell for the `/organization/$slug/people` route. It includes a members table backed by `useListOrganizationMembers`, client-side member search, role filtering, and sorting, permission-gated member role editing and removal for non-current members, self-service leave organization, an `InviteMemberDialog` for inviting members, and an invitations table backed by `useListOrganizationInvitations` with client-side search, filtering, and sorting.

## Usage

```tsx
import { OrganizationPeople } from "@/components/auth/organization/organization-people"

<OrganizationPeople />
```

## Installation

```npm
npx zaidan add https://better-auth-ui.com/r/solid/organization.json
```

## Members

The members section renders a members table with loading and empty states. It can search with a grouped search control, filter by role, and sort members from sortable table headers, opens `InviteMemberDialog` so admins can invite a member with a selected role, lets permitted users update member roles, lets permitted users remove non-current members, and lets the current user leave the active organization.

## Invitations

The invitations section renders an invitations table with loading and empty states. Users can search with a grouped search control, filter, and sort invitations by email, invited date, role, or status from sortable table headers. Users with permission can cancel pending invitations. Advanced status actions remain deferred as a follow-up slice.

## Notes

The People shell exists and is installed with the Organization registry entry. Advanced invitation status actions remain deferred to a follow-up slice.
