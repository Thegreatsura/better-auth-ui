The organization components wire Better Auth's organization plugin into the Solid/Zaidan example. This slice provides slug-driven runtime and profile/settings parity with shadcn for organization routes: `/settings/organizations`, a header switcher, organization-scoped cards, organization profile editing, logo upload, and the route shell at `/organization/$slug/$path` all share the same plugin slug context.

```npm
npx zaidan add https://better-auth-ui.com/r/solid/organization.json
```

## Runtime prerequisites

```ts title="src/lib/auth.ts"
import { organization } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [organization()]
})
```

```ts title="src/lib/auth-client.ts"
import { organizationClient } from "better-auth/client/plugins"

export const authClient = createAuthClient({
  plugins: [organizationClient()]
})
```

## TanStack Start routes

Allow the plugin settings path in `/settings/$path`:

```tsx
import { viewPaths } from "@better-auth-ui/core"
import { organizationPlugin } from "@/lib/auth/organization-plugin"

const validSettingsPaths = [
  ...Object.values(viewPaths.settings),
  ...Object.values(organizationPlugin().viewPaths.settings ?? {})
]
```

Derive the optional route slug in your provider and pass it into the Zaidan organization plugin. Slug routes pass a string; non-slug routes pass `null`.

```tsx title="src/components/providers.tsx"
import { useNavigate, useParams } from "@tanstack/solid-router"
import { organizationPlugin } from "@/lib/auth/organization-plugin"

export function Providers(props: ProvidersProps) {
  const navigate = useNavigate()
  const params = useParams({ strict: false })
  const organizationSlug = () => {
    const slug = params().slug

    if (typeof slug === "string" && slug.length > 0) return slug

    return null
  }

  return (
    <AuthProvider
      navigate={navigate}
      plugins={[organizationPlugin({ slug: organizationSlug() })]}
    >
      {props.children}
    </AuthProvider>
  )
}
```

Add an organization route shell for slug-based organization pages:

```tsx title="src/routes/organization/$slug/$path.tsx"
import { Organization } from "@/components/auth/organization/organization"

function OrganizationPage() {
  const params = Route.useParams()

  return <Organization path={params().path} slug={params().slug} />
}
```

With this setup, `useActiveOrganization` and plugin-contributed organization cards use the same slug-aware plugin context as the route. The switcher and organization rows navigate to `/organization/$slug/$path` in slug-aware mode instead of relying on stale session-active organization state.

## Deferred UI parity

This runtime, create-dialog, and profile/settings slice does not claim the complete copied organization interface. The broader shadcn/HeroUI organization UI still includes members, invitations, role editing, delete dialogs, and leave dialogs. The Solid package APIs for those flows exist, and organization-scoped API key cards are included through the API Key plugin, but the larger copied organization UI remains a follow-up slice to keep PR #393 reviewable.
