## Integration

<Steps>
<Step>
### Configure AuthProvider

Configure `AuthProvider` with TanStack Router's navigation.

<Tabs groupId="shadcn-installation" items={["Registry", "Package"]}>
  <Tab value="Registry">
```tsx title="components/providers.tsx" file=<rootDir>/../../examples/start-shadcn-registry-example/src/components/providers.tsx
```
</Tab>
  <Tab value="Package">
```tsx title="components/providers.tsx" file=<rootDir>/../../examples/start-shadcn-example/src/components/providers.tsx
```
  </Tab>
</Tabs>

The `navigate` and `Link` props integrate with TanStack Router's navigation system. For TanStack Router, you can pass the `navigate` function directly since it accepts `{ to, replace }` options.

</Step>
<Step>
### Update the Root Route

Wrap your application with the `Providers` component in your root route.

```tsx title="routes/__root.tsx" file=<rootDir>/../../examples/start-shadcn-example/src/routes/__root.tsx
```

</Step>
<Step>
### Create the Auth Page

Create a dynamic auth page that renders the appropriate authentication view based on the path.

<Tabs groupId="shadcn-installation" items={["Registry", "Package"]}>
  <Tab value="Registry">
```tsx title="routes/auth/$path.tsx" file=<rootDir>/../../examples/start-shadcn-registry-example/src/routes/auth/$path.tsx
```
</Tab>
  <Tab value="Package">
```tsx title="routes/auth/$path.tsx" file=<rootDir>/../../examples/start-shadcn-example/src/routes/auth/$path.tsx
```
</Tab>
</Tabs>

The `viewPaths.auth` object contains all valid auth paths: `sign-in`, `sign-up`, `sign-out`, `forgot-password`, `reset-password`, and `magic-link`.

</Step>
<Step>
### Create the Settings page

Create a dynamic settings route that renders the `Settings` component for the URL segment. Validate the segment against `viewPaths.settings` so unknown paths return a 404.

<Tabs groupId="shadcn-installation" items={["Registry", "Package"]}>
  <Tab value="Registry">
```tsx title="routes/settings/$path.tsx" file=<rootDir>/../../examples/start-shadcn-registry-example/src/routes/settings/$path.tsx
```
</Tab>
  <Tab value="Package">
```tsx title="routes/settings/$path.tsx" file=<rootDir>/../../examples/start-shadcn-example/src/routes/settings/$path.tsx
```
  </Tab>
</Tabs>

The `viewPaths.settings` object contains valid settings path segments: `account` and `security`.

</Step>
</Steps>

## Protecting Routes

Use the `useAuthenticate` hook to protect routes and access session data.

<Tabs groupId="shadcn-installation" items={["Registry", "Package"]}>
  <Tab value="Registry">
```tsx title="routes/dashboard.tsx" file=<rootDir>/../../examples/start-shadcn-registry-example/src/routes/dashboard.tsx
```
</Tab>
  <Tab value="Package">
```tsx title="routes/dashboard.tsx" file=<rootDir>/../../examples/start-shadcn-example/src/routes/dashboard.tsx
```
</Tab>
</Tabs>

The `useAuthenticate` hook will automatically redirect unauthenticated users to the sign-in page.
