## Prerequisites

Make sure you've completed the [Quick Start](/docs/shadcn) guide first.

## Integration

<Steps>
<Step>
### Configure AuthProvider

Configure `AuthProvider` with Next.js navigation.

```tsx title="components/providers.tsx" file=<rootDir>/../../examples/next-shadcn-example/src/components/providers.tsx
```

The `navigate` and `Link` props integrate with Next.js's navigation system. The `navigate` prop accepts `{ to, replace }` options to handle both push and replace navigation.

</Step>
<Step>
### Update the Root Layout

Wrap your application with the `Providers` component in your root layout.

```tsx title="app/layout.tsx" file=<rootDir>/../../examples/next-shadcn-example/src/app/layout.tsx
```

</Step>
<Step>
### Create the Auth Page

Install the auth components and create a dynamic auth page that renders the appropriate authentication view based on the path.

```npm
npx shadcn@latest add https://better-auth-ui.com/r/auth.json
```

```tsx title="app/auth/[path]/page.tsx" file=<rootDir>/../../examples/next-shadcn-example/src/app/auth/[path]/page.tsx
```

The `viewPaths.auth` object contains all valid auth paths: `sign-in`, `sign-up`, `sign-out`, `forgot-password`, `reset-password`, and `magic-link`.

</Step>
<Step>
### Create the Settings page

If you installed the `settings` component, create a dynamic settings route that renders the `Settings` component for the URL segment. Validate the segment against `viewPaths.settings` so unknown paths return a 404.

```npm
npx shadcn@latest add https://better-auth-ui.com/r/settings.json
```

```tsx title="app/settings/[path]/page.tsx" file=<rootDir>/../../examples/next-shadcn-example/src/app/settings/[path]/page.tsx
```

The `viewPaths.settings` object contains valid settings path segments: `account` and `security`.

</Step>
</Steps>

## Protecting Routes

Use the `useAuthenticate` hook to protect routes and access session data.

```tsx title="app/dashboard/page.tsx" file=<rootDir>/../../examples/next-shadcn-example/src/app/dashboard/page.tsx
```

The `useAuthenticate` hook will automatically redirect unauthenticated users to the sign-in page.
